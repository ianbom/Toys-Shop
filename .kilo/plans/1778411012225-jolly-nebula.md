# Plan: Reload Ongkir Saat Kembali Ke Checkout

## Masalah

- `CheckoutProvider` menginisialisasi `rates` dari `selectedShippingRate ? [selectedShippingRate] : []`.
- `CheckoutScreen` hanya memanggil `loadShippingRates(selectedAddressId)` kalau `shippingRates.length === 0`.
- Saat user kembali ke `/checkout`, context/page menerima `selectedShippingRate` dari server/session, sehingga `shippingRates.length` menjadi `1`.
- Efek reload tidak jalan, UI hanya menampilkan ongkir yang sebelumnya dipilih.

## Solusi

1. Ubah state `rates` di `resources/js/contexts/checkout-context.tsx` agar mulai kosong:
   ```ts
   const [rates, setRates] = useState<ShippingRate[]>([]);
   ```

2. Tetap simpan `currentRate` dari `selectedShippingRate`, agar ringkasan dan tombol bayar masih tahu ongkir terpilih:
   ```ts
   const [currentRate, setCurrentRate] = useState<ShippingRate | null>(selectedShippingRate);
   ```

3. Biarkan `useEffect` di `resources/js/pages/customer/checkout/checkout.tsx` yang sudah ada melakukan fetch daftar ongkir saat mount:
   ```ts
   useEffect(() => {
       if (selectedAddressId && shippingRates.length === 0) {
           void loadShippingRates(selectedAddressId);
       }
   }, [loadShippingRates, selectedAddressId, shippingRates.length]);
   ```

4. Sesuaikan `loadShippingRates` supaya tidak selalu menghapus `currentRate` sebelum daftar baru datang, karena saat halaman dibuka ulang user masih punya ongkir terpilih dari server/session.

5. Setelah daftar ongkir berhasil diambil:
   - Set `rates` dari `payload.rates`.
   - Jika `currentRate` masih valid dan ada di daftar baru, pertahankan pilihan.
   - Jika `currentRate` tidak ada di daftar baru, kosongkan `currentRate` dan reset shipping summary ke `0`.

6. Untuk pemilihan alamat manual (`selectAddress`), tetap reset rate terpilih sebelum fetch, karena alamat berubah berarti ongkir lama tidak valid.

## Implementasi Teknis

- Tambahkan opsi pada `loadShippingRates`, misalnya:
  ```ts
  loadShippingRates(addressId, { preserveSelectedRate = false } = {})
  ```

- `CheckoutScreen` mount call pakai:
  ```ts
  loadShippingRates(selectedAddressId, { preserveSelectedRate: true })
  ```

- `selectAddress` pakai default reset:
  ```ts
  loadShippingRates(addressId)
  ```

## Hasil Diharapkan

- Saat masuk checkout pertama kali, daftar ongkir dimuat dari Biteship.
- Saat keluar checkout lalu balik lagi, daftar ongkir di-fetch ulang, bukan hanya menampilkan satu ongkir lama.
- Ongkir yang sebelumnya dipilih tetap tersorot jika masih tersedia di response terbaru.
- Jika ongkir lama sudah tidak tersedia, pilihan dikosongkan agar user memilih ulang.

## Verifikasi

1. Buka checkout dengan alamat default.

3. Pindah ke profile.
4. Kembali ke checkout.
5. Pastikan daftar ongkir penuh muncul lagi.

7. Pilih alamat lain, pastikan ongkir lama hilang dan daftar ongkir alamat baru dimuat.
8. Jalankan `npm run build` setelah implementasi.
