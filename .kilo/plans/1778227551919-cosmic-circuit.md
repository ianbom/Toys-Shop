# Plan: Navbar Lebih Menyatu Dengan Konten

## Tujuan
Membuat `resources/js/components/Navbar.tsx` terasa lebih menyatu dengan halaman konten/shop layout, bukan seperti blok terpisah. Tetap sticky, tetap readable di atas hero dan halaman produk, tanpa mengubah behavior routing/cart.

## Konteks Ditemukan
- `Navbar` dipakai di `resources/js/layouts/shop-layout.tsx`.
- Background layout utama: `bg-[#f7f0e8]`, text `#53362d`, selection `#e6d5c8`.
- Navbar sekarang: `bg-[#FAF9F6]/90`, padding `py-4`, warna brand/link cukup beda dari layout.
- Homepage hero mulai tepat setelah navbar, jadi navbar harus tetap bisa dibaca di atas konten visual.
- Halaman list/detail pakai `main` dengan `py-6 md:py-10`, jadi navbar height jangan terlalu besar.

## Perubahan Direncanakan
1. Update wrapper `<nav>` di `Navbar.tsx`:
   - Ganti background ke `bg-[#f7f0e8]/78` agar sama dengan layout.
   - Tambah `border-b border-[#dccfc3]/60` supaya ada pemisah halus, bukan panel solid.
   - Pakai `shadow-[0_8px_30px_rgba(83,54,45,0.05)]` sangat tipis.
   - Kurangi padding dari `py-4` ke `py-3 md:py-3.5` agar navbar lebih ringan.
   - Tetap `sticky top-0 z-50 backdrop-blur-xl transition-all duration-300`.

2. Selaraskan warna teks/icon:
   - Brand: `text-[#53362d]` agar sama dengan layout.
   - Link desktop default: `text-[#8b6f63]`.
   - Hover/active visual: `hover:text-[#53362d] hover:border-[#53362d]`.
   - Icon: `text-[#53362d]`, hover `hover:text-[#8b5e4c]`.
   - Badge cart tetap `bg-[#7F2020]` karena sudah kontras dan fungsional.

3. Rapikan desktop nav spacing:
   - Pertahankan struktur existing.
   - Pakai `space-x-8 lg:space-x-10` agar lebih adaptif.
   - Tambah `uppercase` di class container/link bila perlu, sehingga `collection.name.toUpperCase()` bisa tetap atau tidak perlu diubah. Untuk minimal change, tetap gunakan existing uppercase string.

4. Rapikan mobile top nav:
   - Samakan warna dan background wrapper.
   - Brand tetap `font-serif text-2xl`.
   - Icon hover/active tidak mengganggu touch target.

## Verifikasi
- Jalankan format/lint yang tersedia bila script package mendukung, minimal cek TypeScript/ESLint kalau ada script.
- Jika tidak ada script jelas, lakukan review manual `Navbar.tsx` dan pastikan JSX valid.
- Tidak ubah file lain kecuali `Navbar.tsx` saat implementasi.

## Risiko
- Navbar transparan bisa terlihat terlalu tipis pada hero image sangat terang. Mitigasi: opacity `78` plus blur dan border tetap memberi kontras.
- Jika ingin navbar benar-benar overlay hero tanpa mengambil ruang, perlu perubahan layout/positioning lebih besar. Plan ini sengaja minimal dan aman.
